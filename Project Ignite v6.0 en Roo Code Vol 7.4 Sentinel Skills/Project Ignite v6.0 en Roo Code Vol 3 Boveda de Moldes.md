# 📚 Volumen 3: La Bóveda de Plantillas (Sentinel v6.0 — Roo Code)

Este volumen agrupa los 18 moldes maestros utilizados por los motores de Roo Code para garantizar que cada artefacto posea la densidad innegociable y la trazabilidad del "Hilo de Oro".


---

## 🧠 1. El Núcleo de Inteligencia y Gobernanza (Core)
Moldes que rigen el estado, los estándares inamovibles y la visualización del Grafo.

* **project_state_template.json:** El sistema nervioso central. Orquesta fases, veredictos de salud y métricas de metabolismo.
* **project-dna_template.md:** El molde genético. Define los estándares P3/P5 que el auditor usará para certificar el estado `CLEAN`.
* **sentinel-graph-map-template.md:** Molde para la visualización Mermaid que mapea el linaje de todo el conocimiento del proyecto.

---

## 🔍 2. Identidad y Descubrimiento (Discovery Context)
Moldes dedicados a la captura de verdades atómicas y la gestión de la incertidumbre.

* **maestro-seeds.md:** Átomos de verdad (SSoT). Captura el linaje (ID_REF) y gestiona el estado de las semillas de la Tríada.
* **maestro_discovery_log.md:** Diario de cacería forense. Registro secuencial de hallazgos mediante la triple auditoría.
* **maestro_discovery_gap.md:** Mapa de incertidumbre. Registra vacíos de conocimiento y bloquea el avance a la Fase 1.
* **maestro-evolution.md:** Registro de mutación. Documenta colisiones y cambios en el ADN del proyecto tras cada interacción.

---

## 🔱 3. La Pentagonía (Engineering Specs)
Documentación técnica y funcional sincronizada que constituye el `03_knowledge_pack/`.

* **maestro_brd.md:** Visión de Negocio. Ancla políticas (BP) y requerimientos de alto nivel (HLR).
* **maestro_prd.md:** Definición de Producto. Journey del usuario y criterios de aceptación con resiliencia visual.
* **maestro_frd.md:** Lógica Funcional. Máquinas de estado Mermaid y comportamiento determinista.
* **maestro_tech.md:** Plano Técnico. Matriz de los 7 puntos técnicos con blindaje SQL Armor P5.
* **maestro_design.md:** Especificaciones de Diseño. Resiliencia UX P3 y narrativa del flujo visual.
* **maestro_semantic_mapping.md:** Vínculo físico entre semillas de identidad y componentes de la librería UI.

---

## 🔪 4. Ejecución y Metabolismo (Backlog & Interaction)
Moldes para la fragmentación vertical y la traducción de interacciones humanas.

* **maestro-backlog.md:** Cápsula de ejecución. User Story "Dev-Ready" con inyección de código P5 y UX P3.
* **maestro_audit_backlog.md:** Checklist de certificación de calidad antes del paso a desarrollo.
* **meeting-digest-template.md:** Output humano de `/meeting_sync`. Resumen ejecutivo de acuerdos y decisiones.
* **meeting_metabolism_template.md:** Output técnico. Mapea decisiones `NODE-DEC-XXX` y analiza el impacto en el Grafo.
* **decision_log_template.md:** Registro histórico de justificaciones técnicas o de negocio para evitar erosión del conocimiento.
