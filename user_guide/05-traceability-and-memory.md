# Traceability And Memory

Ignite Sentinel separates truth from retrieval.

## Truth Layer

Truth lives in versionable workspace artifacts:

```text
workspaces/[PROJECT_ID]/
```

These files are the authority for requirements, gaps, decisions, specs, backlog, tests, changes, health, and traceability.

## Traceability Layer

The traceability graph lives at:

```text
workspaces/[PROJECT_ID]/06_traceability/traceability_graph.json
```

Human views are generated by:

```powershell
python -m sentinel /trace PROJECT_ID
```

Outputs:

- `traceability_matrix.md`
- `traceability_graph.md`

## Memory Layer

Memory lives at:

```text
workspaces/[PROJECT_ID]/memory.lancedb/
```

Ignite Sentinel uses local LanceDB as the primary retrieval backend:

```text
memory.lancedb/lance/
```

The framework also keeps a JSON hybrid fallback:

```text
memory.json
```

The memory layer stores:

- chunks
- artifacts metadata
- trace edges
- local vectors for hybrid retrieval

It is used by:

```powershell
python -m sentinel /retrieve PROJECT_ID --query "..." --workflow "..."
```

`/ingest`, `/sync`, and `/reindex` populate memory. The indexed workspace context folders are:

- `00_raw/00_client_requirement`
- `00_raw/01_business_context`
- `00_raw/02_technology_context`
- `00_raw/03_design_context`
- `00_raw/04_quality_context`
- `00_raw/05_interactions`
- `07_changes`

These context folders are input owned by the relevant domains. Sentinel retrieves and cites them; it does not replace them with generated tech or design specs.

## Context Packs

Context packs are retrieval snapshots for Codex workflows.

Generate one:

```powershell
python -m sentinel /retrieve PROJECT_ID --query "SLA risk" --workflow sync --write-pack
```

Output:

```text
workspaces/[PROJECT_ID]/08_context_packs/sync.json
```

Use context packs when:

- a workspace is too large to load fully;
- a skill needs only part of the context;
- you want to preserve what context informed a workflow;
- you are reviewing change impact.

## Reindexing

Run reindex after manual edits:

```powershell
python -m sentinel /reindex PROJECT_ID
```

This rebuilds LanceDB memory and the JSON fallback from the graph, versionable artifacts, and context folders.

## Important Rule

If memory and source files disagree, source files win.
